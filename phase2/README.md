# 🚀 Phase 2 – SQL to PySpark (Joins & Aggregations)

## 📌 Objective
In this phase, I focused on applying SQL concepts like joins and aggregations in PySpark using real-world style datasets.

## 💡 My Approach
I first solved each problem using SQL and then converted the same logic into PySpark. This helped me understand how joins and aggregations work in both approaches.

## ✅ Key Learnings
- join() for combining datasets
- groupBy() with sum() and avg()
- filter() for conditions after aggregation
- orderBy() for sorting results

## 🧠 What I Understood
I understood how multiple tables can be combined using joins and how aggregations help in deriving insights like total spend and average values.

## ⚠️ Challenges Faced
- Writing join conditions correctly
- Understanding left join vs inner join
- Applying groupBy with multiple operations

## 🔧 How I Solved Them
- Practiced SQL and PySpark side-by-side
- Broke problems into small steps
- Verified outputs after each query

## 🛠 Tools Used
- DB-Fiddle for SQL
- Spark Playground for PySpark

## 📊 Outcome
I am now able to perform joins and aggregations in PySpark and understand how SQL logic maps to PySpark transformations.
